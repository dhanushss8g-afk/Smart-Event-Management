
# Smart-Event-Management
### Project Overview and Implementation Plan

The **Smart Event Registration + Attendance System** is a modern, high-fidelity web application engineered to run with zero external server or database dependencies. Architected specifically for restricted local environments, it utilizes a custom PowerShell initialization script (`setup_env.ps1`) to download and isolate a portable **Node.js v20.11.1** environment directly within a workspace `.node_env` folder, generating local execution wrappers (`npm.ps1` and `npx.ps1`) to eliminate system pollution. Built on **React 18** and **Vite**, the application avoids heavy utility frameworks like Tailwind CSS in favor of a bespoke, premium **Vanilla CSS Glassmorphism** design system, featuring custom deep-space dark themes, glowing borders, and an interactive particle canvas background component (`<DotField />`). To provide completely functional data persistence without a complex backend installation, all records—including users, events, registrations, attendance rosters, and generated credentials—are managed dynamically using a centralized React context state engine that updates synchronously with the browser's native **LocalStorage**.

The student lifecycle includes a comprehensive, searchable event catalog with live capacity-limit handling, an automated hybrid check-in engine supporting both active camera-based QR code decoding and secondary 6-digit passcode fallbacks via `html5-qrcode`, and a vector-drawn document generation layer using `jsPDF` that renders downloadable, print-ready completion certificates complete with verification IDs and signature lines. For operators, the administrative suite provides high-level telemetry cards for active events, real-time participant manifest tables with manual checking overrides, rapid database schema manipulation tools, and clean CSV data streaming capabilities. To implement this configuration from scratch, an operator must first run the environment script using a process-level execution bypass (`Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`), unpack the package dependency graph using `.\npm.ps1 install`, and spin up the hot-reloading asset pipeline using `.\npm.ps1 run dev`. Evaluation of the deployment is performed by walking through a standard user funnel: initializing a mock registration, verifying remaining capacity adjustments, submitting an automated 6-digit check-in key, downloading the customized credential PDF, and confirming that the administration portal instantly displays updated registry data and registers seamless spreadsheet exports.
